# 让项目完美使用：基于完整真实教材内容生成高质量教案

## 一、Summary（总览）

当前 PDF 教材导入功能已上线（commit a8785b2），但要达到用户期望的"完美使用 + 高质量教案 + 基于完整真实教材内容"，存在四个直接影响质量的缺陷：

1. **PDF 提取的文本完全未清洗**（[app.js:602](file:///workspace/app.js#L602)）：直接 `tc.items.map(it => it.str).join("")`，无段落恢复、无页眉页脚过滤、无断行修正——注入 prompt 的"教材原文"是混乱无换行的长串，LLM 难以理解，是教案质量上不去的最大根源。
2. **没有原文查看/编辑入口**：`idbGetContent` 只在 [app.js:848](file:///workspace/app.js#L848) 生成时调用一次，用户无法看到导入后的原文，更无法修正解析错误或页码映射错误——只能重新导入整本。
3. **prompt 注入未结构化**（[app.js:261-267](file:///workspace/app.js#L261-267)）：原文整段塞进 `【该课时教材原文】`，没有引导 LLM 关注定义/例题/重点，15000 字截断也偏粗暴。
4. **无原文质量自检**：页码填错导致原文与课时不符时，系统照常注入，用户不知情。
5. **扫描版 PDF 无 OCR 兜底**（[app.js:606-608](file:///workspace/app.js#L606-608)）：仅提示"暂不支持"，用户的朋友若拿到扫描版教材会卡住。

本计划一次性补齐五大模块：PDF 原文清洗、OCR 兜底、教材原文管理界面、prompt 结构化注入、生成前原文质量自检。用户已确认选择"完整方案一次到位 + 需要 OCR 兜底"。

```js
// 清洗单页 PDF 文本：恢复段落、过滤页眉页脚、修正断行
function cleanPdfPageText(textItems, pageNum) {
  // 1. 按 transform[5]（y 坐标）分组，同一行内的 item 合并
  //    pdf.js 的 textItem 有 transform 数组，transform[5] 是 y 坐标
  const lines = [];
  let currentY = null;
  let currentLine = [];
  // 按 y 降序排列（PDF 坐标系 y 向上，但getTextContent 返回顺序通常已是阅读顺序）
  const sorted = textItems.slice().sort((a, b) => {
    const ya = a.transform[5];
    const yb = b.transform[5];
    if (Math.abs(ya - yb) > 2) return yb - ya; // y 大的在上
    return a.transform[4] - b.transform[4]; // 同行按 x 升序
  });
  for (const it of sorted) {
    const y = it.transform[5];
    if (currentY === null || Math.abs(y - currentY) <= 2) {
      currentLine.push(it.str);
      currentY = currentY === null ? y : currentY;
    } else {
      if (currentLine.length) lines.push(currentLine.join(""));
      currentLine = [it.str];
      currentY = y;
    }
  }
  if (currentLine.length) lines.push(currentLine.join(""));

  // 2. 断行修正：若行尾不是句末标点（。！？；：.!?）且下一行首字不是引号/括号开头，
  //    则合并到上一行（用空格连接，中文不加空格）
  const merged = [];
  for (let i = 0; i < lines.length; i++) {
    const line = lines[i].trim();
    if (!line) continue;
    const last = merged.length ? merged[merged.length - 1] : null;
    if (last && !/[。！？；：.!?!"'）】》」』]$/.test(last) && !/^["'（【《「『]/.test(line)) {
      // 合并：中文之间不加空格，英文/数字之间加空格
      const sep = /[\u4e00-\u9fa5]$/.test(last) && /^[\u4e00-\u9fa5]/.test(line) ? "" : " ";
      merged[merged.length - 1] = last + sep + line;
    } else {
      merged.push(line);
    }
  }

  // 3. 段落恢复：以句末标点结尾的行视为段落结束，否则视为同一段
  //    实际上经过第2步后，每行已是逻辑行；这里把连续非句末的行合并成段落
  //    （第2步已处理大部分，这里做二次保险：空行作为段落分隔）
  const paragraphs = merged.join("\n").split(/\n\s*\n/).map(p => p.replace(/\n/g, "")).filter(Boolean);

  return paragraphs.join("\n\n");
}

// 跨页页眉页脚检测：统计每页首尾行，找出在 >50% 页面重复出现的行
function detectHeaderFooter(pagesText) {
  const headCount = {};
  const footCount = {};
  const totalPages = pagesText.length;
  for (const text of pagesText) {
    if (!text) continue;
    const lines = text.split("\n");
    if (lines.length > 0) {
      const h = lines[0].trim();
      if (h) headCount[h] = (headCount[h] || 0) + 1;
    }
    if (lines.length > 1) {
      const f = lines[lines.length - 1].trim();
      if (f) footCount[f] = (footCount[f] || 0) + 1;
    }
  }
  const threshold = totalPages * 0.5;
  const headers = Object.keys(headCount).filter(k => headCount[k] >= threshold && k.length < 30);
  const footers = Object.keys(footCount).filter(k => footCount[k] >= threshold && k.length < 30);
  // 页码模式（纯数字或"第X页"）也视为页脚
  return { headers, footers, pagePattern: /^\d{1,3}$|^第\s*\d+\s*页$|^\d+\s*\/\s*\d+$/ };
}

// 从单页文本中移除页眉页脚
function stripHeaderFooter(text, hf) {
  let lines = text.split("\n");
  // 移除首行（若匹配页眉）
  if (lines.length && (hf.headers.includes(lines[0].trim()) || hf.pagePattern.test(lines[0].trim()))) {
    lines = lines.slice(1);
  }
  // 移除末行（若匹配页脚）
  if (lines.length && (hf.footers.includes(lines[lines.length - 1].trim()) || hf.pagePattern.test(lines[lines.length - 1].trim()))) {
    lines = lines.slice(0, -1);
  }
  return lines.join("\n");
}
```

**改造 `handlePdfUpload`**（[app.js:587-616](file:///workspace/app.js#L587-616)）：

```js
// 启发式结构化教材原文：识别定义、例题、正文，分块标注
function structureContent(content) {
  if (!content) return "";
  const paragraphs = content.split(/\n\s*\n/).filter(Boolean);
  const defs = [];      // 定义句
  const examples = [];  // 例题
  const main = [];      // 正文
  for (const p of paragraphs) {
    const trimmed = p.trim();
    // 定义启发式：包含"是""称为""指""叫做""定义"且较短
    if (trimmed.length < 80 && /[称为叫做定义]$/u.test(trimmed) || /^定义[:：]/.test(trimmed) || /就是|是指|叫做|称为/.test(trimmed)) {
      defs.push(trimmed);
    }
    // 例题启发式：以"例""例题""例1""例 1"开头，或包含"解：""解:"
    else if (/^例[\s\d]|^例题|解[:：]/.test(trimmed)) {
      examples.push(trimmed);
    } else {
      main.push(trimmed);
    }
  }
  const parts = [];
  if (defs.length) parts.push("【定义与概念】\n" + defs.join("\n"));
  if (examples.length) parts.push("【例题】\n" + examples.join("\n"));
  if (main.length) parts.push("【正文】\n" + main.join("\n"));
  return parts.join("\n\n");
}
```

```js
// 返回 { ok: boolean, warnings: string[] }
function checkContentQuality(content, lessonTitle) {
  const warnings = [];
  if (!content) {
    return { ok: false, warnings: ["该课时没有教材原文（未导入或页码未填写），将仅凭课时名生成，质量可能较低。"] };
  }
  // 1. 字数过少
  if (content.length < 100) {
    warnings.push(`该课时原文仅 ${content.length} 字，过少。可能页码映射有误，建议到「教材原文管理」检查。`);
  }
  // 2. 关键词匹配：从课时名提取 2 字以上的关键词，检查是否在原文出现
  const keywords = (lessonTitle.match(/[\u4e00-\u9fa5]{2,}/g) || []).filter(k => k.length >= 2 && !["的认识","的应用","练习","复习"].includes(k));
  const missing = keywords.filter(k => !content.includes(k));
  if (keywords.length && missing.length === keywords.length) {
    warnings.push(`课时名关键词（${keywords.join("、")}）均未在原文中出现，可能页码映射错误。`);
  }
  return { ok: warnings.length === 0, warnings };
}
```

绑定 `input` 事件：用户填完页码后，实时调用 `parsePageRange` 计算页码，拼接对应页文本，显示字数。让用户在步骤2就能判断页码填得对不对。

**为什么**：当前要走到步骤3摘要才能看到字数，步骤2填页码时是盲填。实时字数反馈能大幅减少页码映射错误。

## 四、Assumptions & Decisions（假设与决策）

1. **假设 PDF 主体是文本版**：OCR 仅在 pdf.js 提取文本 <50 字时触发，不默认开启，避免拖慢文本版 PDF 的导入速度。
2. **tesseract.js v5.1.0**：选择稳定 UMD 版本，CDN 与 pdf.js 一致（jsdelivr）。语言包 `chi_sim+eng` 首次约 10MB，由 tesseract.js 自动从 CDN 下载。
3. **清洗算法基于启发式规则**：不依赖 LLM 或外部服务，纯 JS 实现。对中文教材效果较好；若用户反馈清洗过度，可在原文管理界面手动修正。
4. **结构化注入用启发式分类**：定义/例题/正文的识别用正则，不追求 100% 准确——目的是给 LLM 一个结构化提示，即使分类有误差，LLM 也能从上下文判断。
5. **原文管理界面不重写整个导入流程**：用户可在管理界面单课时编辑/清空/重新导入（重新导入复用现有向导），但不重做整套向导。
6. **质量自检不阻断**：只 confirm 提示，用户可选择继续，避免误判卡死流程。
7. **不改动 IndexedDB schema**：contentMap 结构不变，仍是 `{key: string}`，清洗后的文本直接覆盖原值。
8. **localStorage 版本号**：`exportAllData` 的版本号从 2 升到 3（含 OCR 标记和清洗后的文本），导入时兼容 v2/v1。
