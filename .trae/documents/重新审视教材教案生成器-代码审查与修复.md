## 三、Proposed Changes（具体修复）

### 修复 1（🔴 关键）：导出/导入保留预置教材的 has_content 标记

**根因**：`getCustomTextbooks()`（[app.js:185](file:///workspace/app.js#L185)）过滤掉 `_override` 项，而 `exportAllData`（[app.js:1623](file:///workspace/app.js#L1623)）用它导出 textbooks。预置教材的 `{id, has_content:true, _override:true}` 覆盖项被排除，导入后 `getAllTextbooks` 找不到 override，预置教材失去 `has_content`，生成时 [app.js:1299](file:///workspace/app.js#L1299) 的 `if (state.selectedTextbook.has_content)` 为假，不加载原文。

**修复方案**：导出时包含 `_override` 项；导入时还原。

**文件**：`/workspace/app.js`

**改动 1**：新增 `getOverrideEntries()` 函数（放在 `getCustomTextbooks` 下方）：
```js
function getOverrideEntries() {
  return lsGet(SK.TEXTBOOKS, []).filter((t) => t._override);
}
```

**改动 2**：`exportAllData`（[app.js:1623](file:///workspace/app.js#L1623)）导出时合并自定义教材 + 覆盖项：
```js
textbooks: getCustomTextbooks().concat(getOverrideEntries()),
```

**改动 3**：`handleImport`（[app.js:1653-1661](file:///workspace/app.js#L1653-1661)）导入 textbooks 时，`_override` 项也要合并（当前逻辑 `!cur.find((t) => t.id === tb.id)` 会跳过已存在的 id，但 _override 项 id 与预置教材相同，需要单独处理）：
```js
if (Array.isArray(data.textbooks)) {
  const cur = lsGet(SK.TEXTBOOKS, []);
  data.textbooks.forEach((tb) => {
    if (!tb || !tb.id) return;
    const existing = cur.find((t) => t.id === tb.id && !!t._override === !!tb._override);
    if (!existing) {
      cur.push(tb);
      merged++;
    } else if (tb._override && tb.has_content) {
      // 更新已有 override 的 has_content
      existing.has_content = true;
    }
  });
  lsSet(SK.TEXTBOOKS, cur);
}
```

**为什么**：这是数据迁移/备份恢复的关键路径。用户导出备份后重装浏览器或换设备导入，预置教材的原文必须可用，否则等于丢失。

### 修复 2（🟡 中等）：checkContentQuality 关键词提取改细

**根因**：`/[\u4e00-\u9fa5]{2,}/g` 匹配连续中文串，"分数的初步认识" 被当作一个关键词，原文中几乎不会完整出现 → 误报。

**文件**：`/workspace/app.js`

**位置**：`checkContentQuality`（[app.js:1262-1263](file:///workspace/app.js#L1262-1263)）

**改动**：关键词提取改为按"的/与/和/及/、/"切分后取 2-4 字片段，并去停用词：
```js
const stopWords = ["认识", "应用", "练习", "复习", "初步", "初步认识", "教学", "学习"];
// 从课时名提取关键词：先按 的/与/和/及/、 切分，再取 2-4 字中文片段
const rawKeywords = lessonTitle
  .split(/[的与和及、，,]/)
  .map((s) => s.trim())
  .filter(Boolean);
const keywords = rawKeywords
  .flatMap((s) => (s.match(/[\u4e00-\u9fa5]{2,4}/g) || []))
  .filter((k) => k.length >= 2 && !stopWords.includes(k));
const uniqueKeywords = [...new Set(keywords)];
const missing = uniqueKeywords.filter((k) => !content.includes(k));
if (uniqueKeywords.length && missing.length === uniqueKeywords.length) {
  warnings.push(`课时名关键词（${uniqueKeywords.join("、")}）均未在原文中出现，可能页码映射错误。`);
}
```

同时移除原有重复的 stopWords 行（[app.js:1261](file:///workspace/app.js#L1261)）。

**为什么**："分数的初步认识" 切分后得到 "分数"、"初步认识"（被停用词过滤），最终关键词为 "分数"，原文中大概率出现。避免误报干扰用户。

### 修复 3（🟡 中等）：structureContent 锚定"解："正则

**文件**：`/workspace/app.js`

**位置**：[app.js:270](file:///workspace/app.js#L270)

**改动**：`解[:：]` 改为 `^解[:：]`：
```js
const isExample = /^例[\s\d]|^例题|^解[:：]/.test(p);
```

**为什么**：原文段落中可能出现"理解：……""了解：……"等，原正则会误判为例题。加 `^` 锚定行首后只有以"解："开头的段落才归为例题。

### 修复 4（🟡 中等）：cleanPdfPageText 段落用 `\n\n` 分隔

**根因**：[app.js:678](file:///workspace/app.js#L678) `return merged.filter(Boolean).join("\n")` 用单 `\n` 连接所有逻辑行，段落间无空行。计划要求段落间 `\n\n`。

**文件**：`/workspace/app.js`

**位置**：`cleanPdfPageText` 第 3 步（[app.js:677-678](file:///workspace/app.js#L677-678)）

**改动**：把"以句末标点结尾的行"视为段落结束，段落间用 `\n\n` 连接：
```js
// 3. 段落恢复：以句末标点结尾的行视为段落结束，段落间用空行分隔
const paragraphs = [];
let cur = "";
for (const line of merged.filter(Boolean)) {
  cur = cur ? cur + line : line;
  if (/[。！？.!?…]$/.test(line)) {
    paragraphs.push(cur);
    cur = "";
  }
}
if (cur) paragraphs.push(cur);
return paragraphs.join("\n\n");
```

**注意**：`cleanOcrText`（[app.js:731-752](file:///workspace/app.js#L731-752)）保持不变——OCR 输出每行通常已是一句，用 `\n` 连接即可，避免过度分段。实际上为保持一致性，也应用相同段落恢复逻辑。**统一处理**：把上述段落恢复逻辑提取为 `mergeIntoParagraphs(lines)` 辅助函数，cleanPdfPageText 和 cleanOcrText 都调用。

**为什么**：段落分明的原文让 LLM 更容易识别定义边界和知识点结构。单 `\n` 连接时，structureContent 虽有 `\n` 分割兜底，但存储/展示/预览的原文可读性差。

### 修复 5（🟢 小）：escapeHtml 增加引号转义

**文件**：`/workspace/app.js`

**位置**：`escapeHtml`（[app.js:32-37](file:///workspace/app.js#L32-37)）

**改动**：增加 `"` 和 `'` 转义（用于 HTML 属性安全）：
```js
function escapeHtml(s) {
  return String(s || "")
    .replace(/&/g, "&amp;")
    .replace(/</g, "&lt;")
    .replace(/>/g, "&gt;")
    .replace(/"/g, "&quot;")
    .replace(/'/g, "&#39;");
}
```

**为什么**：`data-key="${escapeHtml(key)}"`、`data-ch="${escapeHtml(ch.id)}"`、`data-lesson="${escapeHtml(ln)}"` 等属性在课时名含引号时会破损。escapeHtml 全局增加引号转义是最小改动，且对现有 `innerHTML` 内文本内容无副作用（浏览器渲染 `&quot;` 为 `"`）。

**回归检查**：现有用 escapeHtml 的地方都是 HTML 内容或属性，增加引号转义不会破坏显示。
